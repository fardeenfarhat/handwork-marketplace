# Screenshot Specifications for App Stores

## Overview

This document outlines the exact specifications and content requirements for app store screenshots across iOS and Android platforms.

## iOS App Store Screenshots

   - Size: 1242 x 2688 pixels
   - Format: PNG or JPEG

   - Size: 1242 x 2208 pixels
   - Format: PNG or JPEG

## Screenshot Design Guidelines

### Visual Standards

#### Color Scheme
- **Primary**: #007AFF (iOS Blue)
- **Secondary**: #34C759 (Success Green)
- **Accent**: #FF9500 (Warning Orange)
- **Background**: #F2F2F7 (Light Gray)
- **Text**: #000000 (Black), #8E8E93 (Gray)

#### Typography
- **iOS**: San Francisco (System Font)
- **Android**: Roboto
- **Sizes**: 
  - Headers: 24-28px
  - Body: 16-18px
  - Captions: 12-14px

#### Layout
- **Margins**: 16px minimum from edges
- **Spacing**: 8px, 16px, 24px increments
- **Cards**: 8px corner radius
- **Buttons**: 12px corner radius

### Content Guidelines

#### Text Overlay
- **Maximum**: 20% of screenshot area
- **Font**: Bold, high contrast
- **Position**: Top or bottom third
- **Background**: Semi-transparent overlay

#### Sample Data
- **Names**: Use diverse, professional names
- **Locations**: Major cities (New York, Los Angeles, Chicago)
- **Prices**: Realistic market rates
- **Ratings**: 4.0-5.0 range with varied review counts

#### Branding
- **Logo**: Visible but not dominant
- **Colors**: Consistent with brand guidelines
- **Style**: Professional, trustworthy, modern

## Technical Specifications

### File Requirements
- **Format**: PNG (preferred) or JPEG
- **Color Space**: sRGB
- **Compression**: Minimal (high quality)
- **File Size**: Under 10MB per image
- **Naming**: Descriptive (e.g., "ios-home-screen-6.7inch.png")

### Quality Standards
- **Resolution**: Native device resolution
- **Clarity**: Sharp, no pixelation
- **Contrast**: High contrast for readability
- **Lighting**: Even, professional lighting
- **Composition**: Balanced, following rule of thirds

## Screenshot Creation Workflow

### Step 1: Device Setup
1. Use actual devices or high-quality simulators
2. Set up demo accounts with realistic data
3. Configure optimal display settings
4. Clear notifications and status bar clutter

### Step 2: Content Preparation
1. Create sample jobs and worker profiles
2. Generate realistic messages and reviews
3. Set up payment methods and bookings
4. Prepare portfolio images

### Step 3: Capture Process
1. Navigate to target screen
2. Ensure all elements are properly loaded
3. Capture at native resolution
4. Verify image quality immediately
5. Retake if any issues detected

### Step 4: Post-Processing
1. Crop to exact specifications
2. Adjust brightness/contrast if needed
3. Add text overlays (if required)
4. Optimize file size
5. Validate against store requirements

### Step 5: Quality Assurance
1. Review on actual devices
2. Check text readability
3. Verify brand consistency
4. Test with different backgrounds
5. Get stakeholder approval

## A/B Testing Plan

### Metrics to Track
- **Conversion Rate**: App store visits to downloads
- **Click-Through Rate**: Screenshot views to detail page
- **Retention**: Day 1, 7, 30 retention rates
- **User Feedback**: Reviews mentioning screenshots

### Test Variations
1. **Screenshot Order**: Different primary screenshots
2. **Text Overlays**: With vs. without explanatory text
3. **Color Schemes**: Different accent colors
4. **Content Focus**: Worker-focused vs. client-focused

### Implementation
- Use App Store Connect A/B testing
- Google Play Console experiments
- Monitor for 2-4 weeks minimum
- Statistical significance threshold: 95%

This comprehensive screenshot specification ensures high-quality, effective app store listings that accurately represent the Handwork Marketplace app and drive user acquisition.
